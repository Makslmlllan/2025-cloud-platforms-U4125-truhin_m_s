## ITMO University
Faculty: FTMI

## Описание

Laboratory work №2 изучение сервиса Cloud Run в Google Cloud Platform.

## Задание

Ознакомиться с работой Cloud Run, развертыванием сервисов и управлением версиями (revisions).

## Прогресс

### 1. Создание сервиса

Был создан Cloud Run сервис на основе дефолтного контейнера:
gcr.io/cloudrun/hello


Регион: `europe-west1`  
Сервис: `mtrukhin-lab2`


### 2. Тестирование сервиса

После деплоя был получен публичный URL сервиса.  
При переходе отображается сообщение:


It's running!

### 3. Анализ логов

Были изучены логи сервиса.

В логах отображается:
- запуск контейнера
- успешная обработка запросов
- HTTP 200 ответы

### 4. Изменение порта

Был изменён порт приложения с 8080 на 8090.

Результат:
- создана новая ревизия
- сервис продолжил работу (Cloud Run автоматически подставляет PORT)

### 5. Управление трафиком

Трафик был распределён между двумя ревизиями:

- 50% → старая версия
- 50% → новая версия

### 6. Удаление сервиса

После завершения работы сервис был удалён.

Result

В ходе лабораторной работы:

- создан и развернут Cloud Run сервис  
- протестирован endpoint  
- проанализированы логи  
- изменён порт контейнера  
- выполнено управление трафиком между ревизиями  
- сервис удалён  

## Заключение

Cloud Run позволяет быстро разворачивать контейнеризированные приложения, автоматически масштабировать их и управлять версиями без необходимости настройки инфраструктуры.
